# I2CStepper

Скетч для Arduino Nano, управляющий шаговым двигателем и четырьмя реле. Основное назначение - использование совместно с Самоваром как модуль расширения.
Устройство может работать автономно через меню на LCD 16x2 и энкодер или как I2C-slave для Самовара.

Реле:

- `MIXER_PUMP_PIN` - первое реле, обычно насос/мешалка в режиме вкл/выкл.
- `RELE_PIN2`, `RELE_PIN3`, `RELE_PIN4` - дополнительные выходы для периферии.

Режимы шагового двигателя:

- `Mixer` (`I2CMIXER`) - мешалка. Скорость задается в оборотах в минуту, время работы - в секундах. Если время равно `0`, движение продолжается до явной остановки. Для циклической работы доступны пауза и опциональная смена направления после паузы.
- `Pump` (`I2CPUMP`) - перистальтический насос. Скорость задается в мл/ч, движение продолжается до команды остановки. Внешний датчик может остановить насос или поставить его на паузу, если это включено флагами.
- `Fill` (`I2CFILLING`) - дозированный налив через насос. Скорость задается в мл/ч, целевой объем - в мл; после достижения объема шаговик останавливается.

При ручном изменении скорости во время движения цель в шагах не меняется, поэтому отображаемое оставшееся время для `Mixer` или объем для насосных режимов пересчитывается от текущей скорости и оставшихся шагов.

`Mixer` использует роль/адрес `I2CMIXER`, а `Pump` и `Fill` используют роль/адрес `I2CPUMP`. Выбор режима выполняется в меню `Setup` (`Mixer` / `Pump` / `Fill`). После изменения режима или `STP/ML` устройство сохраняет настройки и перезагружается, чтобы Samovar видел корректную роль I2C.

Для насосных режимов `STP/ML` хранит количество шагов на 1 мл. Команды калибровки по I2C рассчитаны на пролив 100 мл: `CALIBRATE_START` запускает насос, `CALIBRATE_FINISH` останавливает его и пересчитывает `STP/ML` как пройденные шаги / 100. Чтобы сохранить результат в EEPROM, после калибровки нужно отправить команду `SAVE`.

При зависании I2C устройство переходит в fail-safe: шаговик останавливается, все реле выключаются. После восстановления шины требуется явное подтверждение кликом энкодера, чтобы снять блокировку.

Начиная с протокола v2 EEPROM хранит `role`, `mode`, настройки Mixer/Pump/Fill, флаги опций, флаги внешнего датчика, маску реле и `STP/ML`. При несовпадении версии EEPROM инициализируется значениями по умолчанию, поэтому после обновления прошивки нужно проверить режим и калибровку насоса.

I2C-протокол v2 описан отдельно: [`docs/i2c-protocol-v2.md`](docs/i2c-protocol-v2.md).

В дальнейшем возможно расширение другими устройствами.

Необходимо использовать библиотеки, идущие в комплекте с Самоваром.

## Сборка через PlatformIO

В репозитории добавлен `platformio.ini`.
Для сборки требуется, чтобы рядом с этим проектом был клон `Samovar` с папкой `libraries`:

- `../Samovar/libraries`

Сборка:

```bash
pio run
```

Подключение периферии:


    // Пины для I2C Master - подключение экрана и потенциально других устройств
    // SDA_PIN 0 //A0
    // SCL_PIN 2 //D2

    // Пины для I2C Slave - связь с Samovar
    // SDA_PIN A4
    // SCL_PIN A5

    // Пины для шагового двигателя
    #define STEPPER_STEP 3
    #define STEPPER_DIR 4
    #define STEPPER_EN 5

    //Пины для реле
    #define MIXER_PUMP_PIN 13 // RELE_PIN1
    #define RELE_PIN2 10      // RELE_PIN2
    #define RELE_PIN3 11      // RELE_PIN3
    #define RELE_PIN4 12      // RELE_PIN4

    // Пины для Encoder
    #define ENC_CLK 7 //S2
    #define ENC_DT 8  //S1
    #define ENC_SW 9  //KEY
